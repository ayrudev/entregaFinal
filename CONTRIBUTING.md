# Práticas de Colaboração e Contribuição

Para garantir a estabilidade do PortfolioHUB, siga o fluxo de trabalho abaixo:

1. Faça um **Fork** do projeto.
2. Crie uma branch para sua nova funcionalidade (`git checkout -b feature/NovaFuncionalidade`).
3. Faça o commit de suas alterações com mensagens claras (`git commit -m 'Adiciona funcionalidade X'`).
4. Faça o Push para a branch (`git push origin feature/NovaFuncionalidade`).
5. Abra um **Pull Request** direcionado à branch `main`.

**Nota de Controle de Acesso:** A branch `main` possui regras de proteção. Nenhum commit direto é permitido; todo código deve passar por revisão (Review) no Pull Request antes do merge.
